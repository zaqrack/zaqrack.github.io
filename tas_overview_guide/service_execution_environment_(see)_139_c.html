Service Execution Environment (SEE)

HPE TAS Service Execution Environment is at the heart of the HPE TAS platform, and may be considered as the macro-kernel of the platform. It is a component that is mandatory in all deployments of HPE TAS. The Service Execution Environment (SEE) is an open Enterprise Java application execution environment (middleware), designed to host services for a wide range of network and communication technologies. It is based on the Wildfly application server and provides security, cluster management, supportability, upgradability, and fault management. The SEE provides real-time access to network elements and facilitates their control.

Each instance of the SEE runs in a JVM and can host one or several applications concurrently, optionally with service orchestration. Each application can be deployed and executed on any of the SEE instances, can use one or more combination of HPE TAS platform APIs, third party APIs, and benefiting from various network & service abstractions.

HPE TAS SEE has several network connectivities which is the integration point of various protocol stacks with HPE TAS SEE.
