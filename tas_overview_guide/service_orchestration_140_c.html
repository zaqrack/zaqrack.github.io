Service orchestration

For simple application requirements, the HPE TAS SEE instance may host a single application directly running on SEE, that is, without the service orchestration layer. However for more complex requirements needing support for multiple applications, the orchestration service must be used. HPE TAS offers a truly transparent orchestration.

HPE TAS offers service-orchestration capability, where the orchestration logic is represented in SCXML. SCXML stands for State-Chart XML, which is a W3C standard. SCXML affords a powerful representation of what may graphically be represented as Harel State-Tables or UML State-Machines.

Using service-orchestration it is possible to blend together a combination of independent applications to create a new behavior which may be seen externally as a new service. This is a powerful way of extending the capabilities of services, of composing new services from other existing service components and of structuring a complex application.

The HPE TAS service-orchestration engine is a high-performance SCXML browser that can not only implement the run-time representation of the state-machine described by the SCXML files it is executing, but also support invocation of fragments of Java code used in SCXML. The HPE TAS SCXML browser implements most of the W3C standard with some limitations and additions.

It is to be noted that service orchestration is not the only way to achieve service interaction and creation of new services.
