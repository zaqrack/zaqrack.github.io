HPE TAS for HPE OCCP and HPE MSE customers

HPE TAS shares a long lineage of products. However, the HPE TAS software is an amalgamation of two very closely related products, namely the HPE Opencall Convergent Communication Platform (HPE OCCP) and the HPE Multimedia Service Environment (HPE MSE).

HPE MSE shares several common components with HPE OCCP and has a greater focus on two specific class of use-cases:

  • Multimedia Control: in close cooperation with HPE OCMP as the media-server or Media Resource Function (MRF)
  • Messaging

HPE MSE focuses on adding capabilities that leveraged and exposed platform capabilities through HTTP-based interfaces (RESTful APIs) to be used by external systems, such as Media-Server (for example, HPE OCMP MRF) or Service Delivery systems (for example, HPE SIS). These HTTP-based interfaces are called connectors. For more information on the supported connectors, Connectors.

HPE MSE also brought certain commonly used, convenience functions:

  • Application Content Manager (ACM) to manage application content such as announcements of audio files, VoiceXML, and CCXML scripts.
  • Tenant Management Portal (TMP) for managing service configuration, subscription in case of multiple tenants used often in multimedia applications.

The HPE MSE shares a large number of common components with HPE OCCP, especially the telecom and related IT protocol stacks (SIP, Diameter, MAP, SMPP, etc.) and common execution environment. HPE OCCP has unique features of its own, such as support for TCAP-based protocols such as CAMEL, INAP and so on, including the option of the Charging gateway application.

HPE MSE (for messaging) and HPE OCCP bring focused Java APIs for session control and for messaging, enabling the development of Java applications that make use of the many telecom and IT protocol stacks integrated in the HPE TAS.

With the aim to strengthen the convergence and enable the capabilities of the two products, the code base of HPE OCCP and HPE MSE are merged in HPE TAS 3.x.
