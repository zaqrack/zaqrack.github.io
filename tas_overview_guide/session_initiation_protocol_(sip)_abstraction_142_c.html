Session Initiation Protocol (SIP) abstraction

The HPE TAS SIP Network Layer provides SIP connectivity and session control for applications.

The HPE TAS SIP Network Layer provides “Third Party Call Control” facilities, where “Third Party Call Control” (in short, 3PCC), is in accordance with the definition in 3GPP TS24.229.

“Third Party Call Control” can be understood as call control handling in an IMS environment within a multimedia session from setup to release. The HPE TAS SIP Network Layer in the context of an IMS environment plays the role of an application server as defined in 3GPP TS24.229. The Application Server role is implemented as a SIP “Back-Back User Agent”.

SIP NL manages the FSM as described by 3GPP standards, while keeping the simplified SCIF call model towards service, manages encode/decode of SIP messages, supports advanced features of SIP like forking, forking masquerading, etc.

Note: For more details on the SIP NL, refer HPE TAS External Functional Specification document.
