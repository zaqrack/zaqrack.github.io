Introduction to HPE TAS

Telecom Application Servers are tasked with the execution of a diverse set of service-plane applications in telecom core networks and thus must meet stringent performance and reliability requirements while affording the use of multitude of communication protocols.

Most Communication Service Provider (CSP) core networks have several SIP Application Servers, NGIN Application Servers (NGIN AS), IN Service Control Points (IN SCP), Interworking Functions (IWF), purpose specific Application Servers such as Conference Servers, Unified Messaging Servers, Unified Communication Servers, Voice-mail Servers, Chat Servers, Presence Servers, and so on.

These Application Servers may share many common traits and functions and support a common set of network protocols. However, many of these Application Servers are closed to third-party development or have their own specific and proprietary application development mechanisms or require their own tooling. The multitude of different Application Servers represents a significant challenge for most CSPs for several reasons, for example:

  • Usage of some Application Servers may decrease or increase over time, leading to under-utilization or strain on the hosting infrastructure.
  • Maintenance of Application Servers can be difficult and expensive over time, due to changing/new regulatory requirements, high infrastructure expense, operations and maintenance.
  • Longer application invocation chain spanning several Application Servers can lead to longer call setup time and poor user-experience.
  • Diversity of Application Servers poses challenges for continued application development and results in slower development and poor innovation agility.

The above challenges can be addressed by consolidating multiple applications onto a single Application Server. HPE Telecom Application Server (HPE TAS) is one such Application Server capable of hosting converged applications that can serve multiple types of networks and a wide range of use-cases. HPE TAS is shortened to HPE TAS in the rest of the document, as part of a product name change, and thus documents refer to the software product as HPE TAS.

With the advent of Voice-and-Video-over-IMS technologies such-as VoLTE, VoWiFi the term TAS has had another expansion in the industry that is Telephony Application Server. It often embodies the Multimedia Telephony (MMtel) function, apart from optionally other functions, such as SCC-AS. MMtel in Telephony Application Server is a specialized application that provides standardized telephony services. However, the 'TAS' in HPE TAS expands to ‘Telecom Application Server’, indicative of its open and more generic applicability for wide range of applications found in telecom networks.
