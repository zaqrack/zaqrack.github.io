Charging Gateway (CGW)

Charging Gateway (CGW) is an important functional element of any Real-time Charging solution. A charging gateway acts as a SIP Application Server (SIP-AS) in an IMS network, or an IN-SCP on legacy PSTN networks or gsmSCF on circuit-switched PLMN networks.

CGW detects occurrences of events and notifies those to the Online Charging Server (OCS) over the standard 3GPP Diameter Ro interface or one of its variants. CGW performs Authorization and Accounting of the service (for example voice-calls, video-calls, short-messages, data-sessions and so on). On successful authorization of the user, OCS responds back to proceed further with accounting / charging of the service.

The HPE TAS Charging Gateway application can fulfill the above described role of a Charging Gateway, while acting as an IN-SCP or gsmSCF for voice, video, fax, CS/PS-data sesions/calls, short-messages in legacy networks using INAP/CAP protocol with the network, and Diameter Ro with OCS.

HPE TAS Charging Gateway is a SCIF service which embeds Diameter Ro Charging Client and interacts with the Diameter Ro server (such as OCS). This can be used to enable real-time charging while rendering various value-added services, implemented as applications in HPE TAS. Charging Gateway can also be used with SCXML-based service-orchestration to achieve service interaction and creation of new services.

Note: For more details on Charging Gateway, see HPE TAS External Functional Specification document.
