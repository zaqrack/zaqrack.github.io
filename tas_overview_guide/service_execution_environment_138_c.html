Service Execution Environment

Applications on HPE TAS are hosted within the HPE TAS - Service Execution Environment (SEE). It embodies the run-time lifecycle of the application and provides the ability to execute the services. The service in execution may be loaded, unloaded, started, stopped and monitored by the HPE TAS SEE.

The administration of the service lifecycle, including configuration, is performed through the Element Management System - Command Line Interface (EMS CLI).

While an application is in execution by HPE TAS SEE, it has access to the network connectivities such as:

  • SIP network connectivity
  • TCAP connectivity
  • DIAMETER connectivity
  • and several others such as SMPP, UCP, and so on

Also, while in execution the application may use the platform common functionalities, such as the following:

  • Application Context
  • Message-sending and receiving which are routed internally by the SEE to appropriate component
  • Application timers
  • Messaging routing and queuing
  • Congestion monitoring and management
  • Logging
  • Number-tracking
  • Statistics update
  • Fault notifications, and so on
Note: Each of these capabilities mentioned above as well as their features, are covered separately In HPE TAS External Functional Specification document.

Each instance of the SEE runs in a JVM and can host one or several applications concurrently, optionally with service orchestration (for more details, see Service orchestration).

Each application can be deployed and executed on any of the SEE instances, meant to use one or more combinations of HPE TAS platform APIs, third party APIs, and benefiting from various network & service abstractions (see Service abstraction).
